# 基于Spring Boot+Vue3的在线考试系统项目规划

### 在线考试系统（综合教学平台）

#### 第一阶段 规划



##### 启动阶段规划



###### 1. 项目背景与总体目标



当前，线上教学在全国范围内迅速普及，各类教育机构及高校对灵活性强、稳定可靠的远程考评工具需求剧增。目前部分日常教学测验仍依赖传统纸质或零散的简易问卷工具，本课题旨在开发一套集中化、前后端分离的计算机考务管理系统，实现教学考评核心数据的数字化。核心目标为提升教师组卷阅卷效率，保障考务数据一致性与安全性，完成从需求分析到系统上线测试的完整系统设计与实现闭环。



###### 2. 系统子系统设计



1. 用户与权限管理子系统 **核心职责**：实现全平台角色的身份验证与结构化管理。 **功能说明**：作为系统基础数据核心，负责全量用户（学生、教师、管理员）的全生命周期管理，维护基础属性与密码安全，建立用户与班级、角色的组织归属关系。 **系统定位**：系统的核心数据存储与访问安全控制中心。

   

   

2. 题库与试卷管理子系统 **核心职责**：对考核资源进行系统化规划与结构化存储。 **功能说明**：支撑考试活动的科学化管理，支持多题型（客观题、主观题）录入与分类标签化；提供手动组卷与基于规则的自动随机抽题功能，为教师团队提供考核内容构建的结构化工具。 **系统定位**：考核资源的量化存储与试卷生成引擎。

   

   

3. 考试过程监控子系统 **核心职责**：实现所有线上测验过程的标准化与数据归档。 **功能说明**：提供考试全流程管理，覆盖考前发布、在线倒计时答题、防作弊切屏检测、自动交卷等功能，保障答题数据的完整性、准确性与时效性。 **系统定位**：官方的考试环境承载与过程归档平台。

   

   

4. 阅卷与数据可视化子系统 (基于ECharts) **核心职责**：持续计算并结构化评估学生的考试成绩指标。 **功能说明**：客观题自动判卷，主观题提供教师批阅界面。利用ECharts图表引擎，将复杂的考试结果转化为直观的可视化图表展示（如成绩分布柱状图、知识点掌握雷达图、及格率饼图），生成班级与个人的多维度分析报告。 **系统定位**：学情数据的量化评估与可视化展现中枢。

   

   

###### 3. 核心用户群体与功能需求



系统面向三类核心角色，职责与功能需求明确区分：



**A. 教师（考务人员）**





- 题库与试卷管理：对所属科目的题库进行新增与维护，设定规则生成试卷；

  

  

- 考试管理：记录并发布考试日期、时长、及格线及参与班级；

  

  

- 阅卷分析：查阅主观题并打分，通过ECharts图表查看班级整体成绩分布。

**B. 管理员**





- 中央用户登记：维护全校/机构全体师生的安全数据库，支撑系统合规运行；

  

  

- 系统配置维护：管理基础数据字典（如科目分类、角色权限分配）。

  

  

**C. 学生（考生）**





- 在线答题：查看待考列表，在规定时间内完成作答并提交试卷；
- 成绩查询：查看个人历史考试记录、成绩得分及错题解析。

###### 4. 非功能性需求



- **易用性**：前端采用Element-Plus组件库，界面直观简洁；适配不同终端设备，减少用户的学习成本。

  

  

- **数据一致性**：客观题自动判分与总成绩汇总实时同步，无冗余、无冲突。

  

  

- **可靠性**：考试作答期间支持状态缓存，在网络波动等关键场景中杜绝答题数据丢失。

  

  

###### 5. 软硬件及知识资源配置



本项目资源规划明确，采用现代化的前后端分离技术栈：

**（1）核心技术栈与软件资源**

- **数据库**：采用 MySQL 8.4 进行数据存储（开发环境账号密码已配置为：`root`, `962464`）。
- **前端工程**：位置严格限定于 `/src/main/resources/frontend` 目录下；采用 Vite 作为构建工具，搭配 Vue3 (Composition API) 与 JavaScript 进行视图开发；依赖包管理采用 `npm`；UI 框架选用 Element-Plus；图表展示集成 ECharts 库。

**（2）知识资源**





- Spring Boot/Vue3 开发文档：掌握 RESTful API 规范、Vue 响应式原理及组件化开发。

###### 6. 确定开发的三大核心子系统说明



结合项目体量，优先落地以下三大核心子系统：



1. **题库与试卷管理子系统**：全面管控考试核心资源，支撑自动/手动组卷业务逻辑；

   

   

2. **考试过程监控子系统**：完成所有考试相关记录与并发答题处理，实现考试全生命周期管理；

   

   

3. **阅卷与数据可视化子系统**：依托ECharts组件，实现成绩的统计、呈现及学情追踪。

   

   

###### 7. 项目目标



本项目目标为设计并开发前后端分离架构的考试系统原型，提供数字化考评解决方案。具体目标如下：



1. 开发高质量系统原型，落地三大核心子系统功能；

   

   

2. 将传统的线下测验转化为集中化线上流程，提升师生效率；

   

   

3. 通过统一的API接口和Vue3前端，为师生提供流畅的单页面应用(SPA)体验；

4. 通过丰富的ECharts图表，为教师提供结构化的成绩查询渠道，辅助教学决策。

###### 9. 开发方法论



本项目采用**瀑布开发模型**（结合敏捷原型迭代），该模型为软件开发提供清晰、结构化的顺序化流程。划分为**需求分析、系统设计（含数据库与接口定义）、前后端开发实现、系统测试、系统部署**五大阶段。



- **选用原因**：课题需求清晰稳定；各阶段需形成正式文档，便于后续撰写总结报告与文档；前后端分离架构下，优先定义好接口文档（Swagger/Apifox）即可并行开发，流程逻辑连贯。

#### 第二部分 解决方案规划



##### 在线考试系统子系统详情



###### 1. 题库与试卷管理子系统



**（1）核心功能模块**





- **题目 CRUD 模块**：支持对五大基本题型的增删改查，所有数据通过 MyBatis-Plus 映射至 MySQL 底层存储。
- **智能组卷模块**：控制器接收前端表单的抽题规则（题型、数量、难度），调用业务逻辑层在题库中进行随机检索并绑定生成试卷实体。

**（2）数据关系与处理流程**





- **数据关系**：试卷与题目为**多对多关系**（一张试卷含多道题，一道题可出现在多张试卷），需设计中间关联表（如 `paper_question`）记录关联及此题在本试卷中的设定分值。

  

  

###### 2. 考试过程监控子系统



**（1）核心功能模块**





- **考务发布模块**：记录考试唯一标识、开考时间、持续时长及状态。

  

  

- **答题会话模块**：学生进入考试后，前端 Vue3 触发倒计时组件；考试中途前端实时将作答快照发往 Spring Boot 后端进行 Redis 或数据库缓存，防数据丢失。

**（2）核心业务流程**





考试创建（记录基础信息） → 考生进入（验证时效与权限） → 动态答题（防切屏监控） → 自动/主动交卷（锁定数据，流转至阅卷系统）。



###### 3. 阅卷与数据可视化子系统



**（1）核心功能模块**





- **自动核对模块**：客观题比对数据库标准答案，直接生成分数。
- **可视化看板模块（核心集成 ECharts）**：基于前端 `/src/main/resources/frontend` 目录下的 Vue3 组件，引入 ECharts 实例。通过 Axios 向 Spring Boot 请求统计数据（JSON格式）。

**（2）图表数据分析展示**





系统自动生成成绩柱状分布图、班级平均分折线趋势图、错题率排行的横向条形图。为教师评估教学效果提供直观、客观的依据。



##### 界面（UI）设计原则与集成方案



系统UI基于前端 Vite+Vue3 架构设计，大量运用 Element-Plus 组件库，设计理念围绕**易用性、现代感**：



1. **视觉与功能一致性**：采用 Element-Plus 的默认主题或定制化主题变量，侧边菜单栏采用 `<el-menu>` 组件固定显示，避免操作迷失。

   

   

2. **简约美学设计**：大量采用卡片式布局 (`<el-card>`)，将信息划分为简洁模块；数据列表采用 `<el-table>` 结合分页组件 (`<el-pagination>`)。

   

   

3. **及时反馈**：全面应用 Element-Plus 的 Message 消息提示和 Notification 通知组件，任何数据的 CRUD 操作完成后即时给出视觉反馈（如“交卷成功”）。

   

   

4. **图表无缝嵌入**：ECharts 图表容器按需分配固定宽高，并监听窗口 `resize` 事件实现响应式缩放，确保在数据大屏或普通笔记本视图下均能完美呈现。
