# 在线考试系统主要功能模块

整理依据：后端控制器 `src/main/java/com/ekusys/exam/*/controller`、业务服务 `src/main/java/com/ekusys/exam/*/service`、实体映射 `src/main/java/com/ekusys/exam/repository/entity`、数据库脚本 `src/main/resources/schema.sql` 与 `src/main/resources/db/migration`。

| 模块名称 | 涉及控制器 | 涉及服务类 | 涉及数据库表 | 功能说明 |
| --- | --- | --- | --- | --- |
| 用户认证与会话安全模块 | `AuthController` | `AuthService`、`AuthCookieService`、`RefreshTokenSessionService`、`CustomUserDetailsService`、`JwtTokenProvider`、`AuthRateLimitService` | `sys_user`、`sys_role`、`sys_user_role` | 提供登录、刷新令牌、退出登录、当前用户信息查询等能力。系统通过用户名密码校验用户身份，签发 JWT 访问令牌，并通过刷新令牌维持会话；同时加载用户角色权限，支持账号启用状态校验与登录频率限制。 |
| 系统基础数据管理模块 | `AdminController` | `AdminService`、`UserAdminService`、`RoleAdminService`、`SubjectAdminService`、`TeachingClassAdminService`、`UserProfileSyncService` | `sys_user`、`sys_role`、`sys_user_role`、`subject`、`student_profile`、`teacher_profile`、`teaching_class`、`student_teaching_class` | 面向管理员提供用户、角色、课程、教学班等基础数据维护功能，包括用户分页查询、新增、修改、删除、重置密码、分配角色，课程新增/修改，教学班新增/修改等。用户角色变化时同步维护学生档案、教师档案和学生选课关系。 |
| 管理端批量导入与批量操作模块 | `AdminController` | `AdminBulkService`、`AdminCsvImportService`、`UserAdminService`、`RoleAdminService`、`TeachingClassAdminService`、`ExamService` | `sys_user`、`sys_role`、`sys_user_role`、`student_profile`、`teacher_profile`、`teaching_class`、`student_teaching_class`、`subject`、`paper`、`exam`、`exam_target_class` | 支持通过 CSV 文件批量导入用户、教学班和考试安排，并提供 `dryRun` 预校验能力；同时支持用户、教学班和考试的批量启用、停用、删除或状态类操作，用于降低管理员维护成本。 |
| 教师班级与学生管理模块 | `TeacherClassController` | `TeacherClassService` | `teaching_class`、`student_teaching_class`、`student_profile`、`sys_user`、`sys_role`、`subject` | 面向教师提供本人教学班查询、班级学生列表查询、可加入学生候选查询、添加学生到教学班、从教学班移除学生等功能。系统会校验教师对班级的管理权限，并维护学生与课程/教学班之间的关联关系。 |
| 题库与题目资源管理模块 | `QuestionController` | `QuestionService`、`QuestionImageService`、`QuestionAssetUrlResolver`、`QuestionAssetCleanupService`、`QuestionAssetCleanupScheduler` | `question`、`question_asset`、`subject`、`paper_question` | 面向教师和管理员提供题目分页查询、课程选项查询、题目详情、新增、修改、删除等题库管理能力。题目支持题型、难度、选项、答案、解析和默认分值维护；题图通过 MinIO 上传并记录到 `question_asset`，后台定时清理未绑定或无效资源；已被试卷引用的题目会受到删除限制。 |
| 试卷与组卷管理模块 | `PaperController` | `PaperService` | `paper`、`paper_question`、`question`、`question_asset`、`subject`、`exam` | 提供手动组卷、自动组卷、试卷分页查询、试卷详情、试卷修改和删除功能。手动组卷按教师选择的题目及分值生成试卷；自动组卷按题型、难度、数量等规则从题库抽题；系统维护试卷总分和题目排序，并在试卷已被考试使用时限制删除。 |
| 考试发布与考试过程模块 | `ExamController` | `ExamService`、`ExamLifecycleService`、`ExamTeacherQueryService`、`ExamStudentQueryService`、`ExamStartService`、`ExamSessionService`、`ExamSnapshotService`、`ExamSubmissionService`、`ExamSubmissionAcceptanceService`、`ExamSubmissionProcessingService`、`ExamSubmissionMessagePublisher`、`ExamSubmissionConsumer`、`ExamQuestionAssembler`、`ExamAutoSubmitService`、`ExamAutoSubmitScheduler`、`ExamStatusService`、`ExamAccessService`、`ExamPermissionService`、`SnapshotFlushScheduler` | `exam`、`exam_target_class`、`exam_session`、`submission`、`submission_answer`、`paper`、`paper_question`、`question`、`question_asset`、`teaching_class`、`student_teaching_class` | 覆盖考试创建、发布、终止、教师考试列表、学生可参加考试列表、学生成绩列表、考试开始、答题快照保存、交卷和超时自动交卷等核心流程。系统按目标教学班控制考试范围，按考试时间刷新状态，学生开始考试后生成答题会话和截止时间；答题草稿先进入快照缓存并定时落库，正式交卷后生成提交记录和答案明细，客观题自动判分，主观题进入待阅卷状态；提交处理支持 RabbitMQ 异步消费。 |
| 在线监考与反作弊模块 | `ExamController` | `ExamProctoringService`、`ExamAntiCheatWriteService`、`ExamAntiCheatEvidenceService`、`ExamProctoringPolicyService`、`ExamAccessService`、`ExamPermissionService` | `anti_cheat_event`、`proctoring_disposition`、`exam`、`exam_session`、`submission`、`exam_target_class`、`teaching_class`、`student_teaching_class`、`sys_user` | 支持考试监考概览、监考学生列表、单个学生监考时间线、异常处置记录维护、学生端反作弊事件上报和反作弊证据上传。考试可配置监考等级和策略，系统根据策略决定是否记录事件；异常事件写入 `anti_cheat_event`，证据文件上传到 MinIO 后以 JSON 形式关联事件，教师或管理员可对学生异常情况进行处置并保存到 `proctoring_disposition`。 |
| 阅卷评分模块 | `GradingController` | `GradingService` | `submission`、`submission_answer`、`subjective_grade`、`paper_question`、`question`、`exam`、`sys_user` | 面向教师和管理员提供待批阅答案列表、按题目分组的待批阅列表、题目下待批阅学生答案查询、单份提交主观题评分、按题目批量评分等功能。评分时会校验题型和分值范围，记录教师评分明细，并重新计算提交的主观分、总分和及格状态。 |
| 成绩统计与学情分析模块 | `AnalyticsController` | `AnalyticsService` | `submission`、`submission_answer`、`exam`、`paper`、`student_teaching_class`、`teaching_class`、`question` | 为教师和管理员提供考试成绩分布、考试概览、班级趋势和错题排行等分析功能。系统基于提交记录、答案明细、班级关系和题目信息聚合统计，可用于查看考试整体通过情况、班级表现差异和高频错误题目。 |
| 管理端考试监控模块 | `AdminController` | `AdminExamMonitorService`、`ExamStatusService` | `exam`、`exam_target_class`、`exam_session`、`submission`、`anti_cheat_event`、`teaching_class`、`student_teaching_class`、`sys_user` | 面向管理员提供考试监控汇总和指定考试学生状态查看能力。系统可按时间范围统计考试数量、进行中考试、参考学生状态、提交情况和反作弊事件数量，并支持查看某场考试内每个学生的答题会话、提交状态及异常事件概况。 |
| 操作审计日志模块 | `AdminController` | `OperationAuditLogService`、`OperationAuditLogQueryService`、`OperationAuditLogAspect` | `operation_audit_log` | 对用户管理、角色分配、题库、试卷、考试发布/终止、阅卷、监考处置、批量导入等关键操作进行 AOP 审计记录，保存操作人、角色、请求方式、请求路径、IP、目标对象、操作状态、错误信息和操作时间。管理员可按操作人、动作、目标类型、状态和时间范围分页查询审计日志。 |
| 公共支撑与系统配置模块 | `GlobalExceptionHandler`（全局异常处理） | `SecurityConfig`、`MinioConfig`、`RabbitMqConfig`、`MybatisPlusConfig`、`MybatisMetaHandler`、`LegacySchemaCompatibilityInitializer`、`JacksonConfig` 等配置类 | 主要支撑全系统表结构和外部组件访问，不单独对应业务表 | 提供全局安全过滤链、JWT 认证过滤、跨域配置、统一异常响应、MyBatis-Plus 审计字段填充、MinIO 对象存储配置、RabbitMQ 提交消息配置、JSON 序列化配置以及旧表结构兼容初始化等基础能力，是各业务模块稳定运行的公共基础。 |
